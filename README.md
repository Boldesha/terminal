# terminal
##### Программа выполняющая работу терминала с собственной виртуальной файловой системой. Не взаимодействует с реальной ОС: все сущности и данные хранятся в оперативной памяти. Главный фокус работы — демонстрация полиморфизма и плоской иерархии классов.

### Команды:
1. help - Вывод списка доступных команд.
2. exit - Завершить работу.
3. ls - Вывод дочерних сущностей текущей директории (скрытые фильтруются через isHidden()).
4. cd <name|..> - Переход в поддиректорию или на уровень выше (защита от выхода выше root).
5. cat <name> - Вывод содержимого файла.
6. touch <name> - Создание нового текстового файла с выбором типа (многострочный или однострочный) в текущей директории.
7. mkdir <name> - Создание новой директивы в текущем каталоге.
8. rm <name> - Скрытие сущности (мягкое удаление).
9. mv <old> <new> - Переименование любой сущности (файла или папки) с old на new.
10. restore <name> - Возврат скрытой сущности в список.
11. edit <name> - Редактирование файлов.

### Типы файлов:
1. SingleLineFile - однострочные текстовые файлы.
2. MultiLineFile - многострочные текстовые файлы.

### Иерархия классов:
1. Entry - Корневой базовый класс для всех сущностей ФС. Хранит name и hidden. Содержит виртуальные методы (show, edit, cat, hide, restore, isDirectory, asDirectory) с реализацией по умолчанию. Не абстрактный (по требованию задания).
2. Directory - Папка / узел дерева. Наследуется напрямую от Entry. Хранит vector<Entry*> дочерних элементов и ссылку на parent. Переопределяет методы для списка детей, поиска и навигации.
3. SingleLineFile - Однострочный текстовый файл. Наследуется от Entry. Хранит std::string content. Переопределяет edit() (замена строки) и cat() (вывод).
4. MultiLineFile - Многострочный текстовый файл. Наследуется от Entry. Хранит std::vector<std::string> lines. Переопределяет edit() (изменение по индексу + добавление новых строк) и cat().
5. Terminal - Контроллер приложения (вне иерархии Entry). Управляет циклом ввода, парсит команды, хранит root и current. Делегирует все действия объектам через Entry*.

Плоская иерархия: Directory, SingleLineFile, MultiLineFile наследуются напрямую от Entry.
Полиморфизм вместо if/else: Terminal работает только с Entry*. Поведение различается исключительно через переопределённые virtual методы (e->edit(), e->cat() и т.д.).
Без абстракций и интерфейсов
Terminal управляет вводом и навигацией, Entry и его наследники инкапсулируют состояние и логику сущностей. Данные хранятся в памяти, реальная ФС ОС не затрагивается.

## Build
make all

## Run
./terminal
